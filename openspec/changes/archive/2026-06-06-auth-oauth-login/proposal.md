# 提案：OAuth登录/注册功能

## 问题背景

当前 Lingua Journey 项目只有基础的邮箱/密码注册和登录功能，缺少现代用户友好的 OAuth 登录方式，这会降低用户的注册转化率和使用体验。

## 目标

1. 支持通过 **Google OAuth** 登录/注册
2. 支持通过 **GitHub OAuth** 登录/注册
3. 完善 **邮箱验证码** 登录/注册
4. 提供美观的前端登录/注册界面
5. 个人资料管理页面
6. 支持多种登录方式统一账户关联

## 范围

### 包含内容

**后端功能：
- OAuth 2.0 集成（Google、GitHub）
- 邮箱验证码发送与验证
- 用户 OAuth 账户关联管理
- OAuth 回调处理
- 统一的认证接口

**前端功能：**
- 登录页面（邮箱登录、Google 登录、GitHub 登录）
- 注册页面
- 个人资料页面
- 账户设置页面（关联/解绑 OAuth 账户）
- 找回密码功能

### 不包含内容

- 其他第三方登录（Facebook、Twitter 等，后续可扩展）
- 手机号登录（后续可扩展）
- 双因素认证（后续可扩展）

## 成功标准

- 用户可以通过 Google、GitHub 或邮箱验证码三种方式注册/登录
- OAuth 登录流程顺畅无 bug
- 前端界面美观、响应式
- 安全符合安全最佳实践
