## Context

学习路径推荐系统旨在为用户提供个性化的学习路线图。当前Lingua Journey平台用户完成课程后缺乏下一步学习方向。通过分析用户的学习目标、当前水平、已完成课程和学习偏好，系统将生成定制化的学习路径推荐。

## Goals / Non-Goals

**Goals:**
- 提供清晰的学习目标设置和管理功能
- 根据用户目标和水平生成个性化学习路径
- 智能推荐适合用户的课程
- 可视化展示学习路径进度
- 支持路径动态调整

**Non-Goals:**
- 不实现复杂的AI推荐算法（第一版本使用规则引擎）
- 不实现课程自动跳过或免学功能
- 不实现多人协作学习路径

## Decisions

### 1. 学习目标设计

**方案选择：** 独立目标表 + 目标类型枚举

```
LearningGoal (目标表)
├── id, user_id, title, description
├── goal_type (短期/中期/长期)
├── target_date (目标完成日期)
├── status (进行中/已完成/已放弃)
├── created_at, updated_at

LearningPath (路径表)
├── id, user_id, goal_id
├── course_order (课程顺序数组)
├── current_position (当前位置索引)
├── status (active/completed/abandoned)
├── generated_at, updated_at
```

### 2. 推荐算法设计

**方案选择：** 规则引擎 + 协同过滤简化版

推荐因素权重：
- 用户目标语言匹配：30%
- 用户当前水平匹配：25%
- 已完成课程补全：20%
- 热门课程加权：15%
- 用户偏好标签：10%

**替代方案考虑：**
- 机器学习推荐：需要大量数据和训练资源，暂不采用
- 纯热门推荐：缺乏个性化

### 3. 路径生成策略

**方案选择：** 拓扑排序 + 难度递进

- 课程间存在先修关系时，按拓扑排序
- 无先修关系时，按难度级别递进排列
- 优先推荐用户未学习但与目标相关的课程

### 4. 课程推荐展示

**方案选择：** 分类展示 + 个性化推荐理由

- "为你推荐"区：基于当前路径的下一课程
- "热门课程"区：平台热门课程
- "根据你的目标"区：与用户学习目标匹配的课程

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 推荐算法不够精准 | 第一版使用规则引擎，逐步收集数据优化 |
| 课程数据不足导致推荐质量差 | 提供"手动选择课程"备选方案 |
| 用户目标设置过于模糊 | 提供预设目标模板辅助用户 |
| 路径过长导致用户放弃 | 支持路径分段，优先完成短期目标 |

## Open Questions

1. 推荐算法权重是否需要用户可配置？
2. 学习路径是否需要支持多人共享/协作？
3. 是否需要集成外部课程资源扩展推荐池？
